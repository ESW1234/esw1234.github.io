TC02: Slow 3G Download of a 5MB PDF Succeeds First Try (Customer Repro)

This is the core bug scenario from W-22247030. The original customer (Copyright Clearance Center) saw downloads abort at 862–950ms due to the 1000ms iframe cleanup timeout.

Pre-conditions

  • Chrome DevTools open (F12)
  • Network throttling set to Slow 3G (DevTools → Network → Throttling dropdown)
  • Active live-agent session with file transfer capability
  • Agent has a ~5MB PDF ready to send

Steps to Reproduce

  1. Open Chrome DevTools (F12) and go to the Network tab.
  2. Set throttling to Slow 3G.
  3. Open this page and wait for the chat widget to appear.
  4. Click the chat FAB to start a conversation.
  5. Ask the agent to send a ~5MB file attachment.
  6. When the file attachment appears, click the download button.
  7. Observe the Network panel — watch for the iframe request timing.
  8. Capture HAR if needed: right-click in Network panel → Save all as HAR.

Expected Results (Post-Fix)

  • Download completes on first click — no retry needed.
  • HAR shows the request reaching commit even if total time exceeds 1000ms.
  • No net::ERR_ABORTED in the Network panel.
  • File content matches source byte-for-byte.

Pre-Fix Behavior (Bug)

  • Download silently fails or stalls at ~862–950ms.
  • Network panel shows (canceled) on the iframe request.
  • Retrying sometimes succeeds (race condition with the 1000ms timeout).
