Accessibility Regression Manual

TC06: Hidden Download Iframe Stays Out of Focus Order and A11y Tree

Pre-conditions

  • Active session with the agent
  • Chrome DevTools open (Elements panel + Accessibility panel)
  • Optional: VoiceOver (macOS) or NVDA (Windows) running

Steps to Reproduce

  1. Open this page and start a chat session.
  2. Ask the agent to send a file attachment.
  3. Open Chrome DevTools → Elements → Accessibility panel (sidebar).
  4. Click the download button on the attachment.
  5. While the download is in-flight, inspect the accessibility tree for any new iframe elements:
// Check a11y tree for hidden iframes document.querySelectorAll('iframe[style*="display: none"]').forEach(el => { console.log('Hidden iframe:', el.title, '| tabIndex:', el.tabIndex, '| aria-hidden:', el.getAttribute('aria-hidden')); });
  1. Try pressing Tab repeatedly through the page. Check if focus ever lands on the hidden iframe.
  2. If using a screen reader, listen for any announcement of a new iframe during the download.
  3. Trigger 5 downloads in a row and listen for SR announcements each time.

Expected Results

  • Iframe has style="display: none" — does not appear in the a11y tree.
  • Tab key never focuses the hidden download iframe.
  • Screen reader does not announce a new element on each download.
  • The iframe title attribute is present (e.g. "File Download") for cases where it might become visible.
  • Existing SR announcements for the messaging surface are unchanged.
