Error Handling & Recoverability Manual

TC05: Network Drops Mid-Download

Pre-conditions

  • Active session with the agent
  • A large file (~10MB+) ready to be sent by agent (to keep download in-flight long enough)
  • DevTools open with Network tab and Console visible

Steps to Reproduce

  1. Open this page and start a chat session.
  2. Ask the agent to send a large file attachment (~10MB).
  3. Click the download button on the attachment.
  4. While the download is in-flight (visible in Network panel as pending), toggle the browser to Offline mode:
    DevTools → Network → check "Offline" checkbox.
  5. Observe the Console for errors.
  6. Toggle back Online (uncheck "Offline").
  7. Click the download button again to retry.

Expected Results

  • First attempt surfaces a clear failure (event/log in console) — NOT a silent abort.
  • The iframe is still cleaned up (no orphan iframe after failure).
  • Retry after reconnect succeeds — file downloads completely.
  • No uncaught exceptions that break subsequent downloads.
  • No removeChild-on-detached-node error in console.
